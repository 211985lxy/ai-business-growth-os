### Phase 2：架构解耦 + 第二个模块（2-4周）

> 核心目标：为其他五个模块铺路，建立模块间数据流通

**2.1 提取通用三栏布局**
```
components/layout/three-column-layout.tsx
hooks/use-panel-collapse.ts
```
让所有模块共享同一套布局逻辑，新模块开发只需要：
- 定义左侧面板内容
- 定义输入表单
- 定义输出渲染器

**2.2 建立跨模块上下文**

Supabase 新建 `shared_context` 表：
```sql
create table shared_context (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users(id),
  context_type text not null,     -- 'niche', 'audience', 'brand_voice', etc.
  context_key text not null,      -- 可检索的键
  context_value jsonb not null,   -- 结构化数据
  source_agent text not null,     -- 哪个智能体产出的
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);
```

战略模块生成完成后，自动提取关键信息写入 shared_context：
- 赛道定位 → context_type='niche'
- 目标用户 → context_type='audience'
- 竞争优势 → context_type='strengths'
- 品牌调性 → context_type='brand_voice'

内容模块启动时，自动读取这些上下文注入提示词。

**2.3 启动神韵·内容模块**

基于通用三栏布局，开发内容模块：
- 左侧：内容历史 + 平台选择（小红书/抖音/公众号）
- 中间：内容参数（类型/调性/关键词）+ 自动加载战略上下文
- 右侧：AI生成的内容 + 多平台预览

**2.4 模型升级路径**

通过硅基流动按需切换模型，不改Dify架构：
```
Dify Chatflow → LLM节点配置硅基流动API → 按模块选择最优模型
```

| 模块 | 推荐模型 | 原因 |
|------|---------|------|
| 天道·战略 | DeepSeek-V3.2 / GLM-4.6 | 推理分析能力强 |
| 神韵·内容 | GLM-4.6 / Qwen2.5-72B | 中文创作质量高 |
| 法度·风控 | Qwen2.5-7B (免费) | 规则匹配为主，不需要大模型 |

### Phase 3：OpenClaw集成 + 多渠道（4-8周）

> 核心目标：从"打开网页生成报告"升级到"发条消息自动执行"

**3.1 部署OpenClaw**
- 在Mac Mini或VPS上部署OpenClaw实例
- 学习Skill开发，先写一个hello-world Skill跑通

**3.2 开发六脉Skills**

每个"脉"对应一个OpenClaw Skill，代码里已预留接口：
```typescript
// lib/dify/client.ts 末尾已预留
export async function callAgentSkill(
  agentType: "strategy" | "content" | "earth" | "man" | "law" | "money",
  input: Record<string, unknown>
): Promise<void> {
  // Phase 3 在此实现
}
```

Skill之间通过OpenClaw Memory共享上下文，替代Phase 2的Supabase shared_context方案（或两者并存）。

**3.3 接入消息渠道**
- WhatsApp（OpenClaw原生支持）
- Telegram（OpenClaw原生支持）
- 微信（需要额外开发适配层）

**3.4 自动化任务**
- 每日竞品监控 → 推送摘要到用户
- 每周增长报告 → 自动生成并发送
- 内容日历提醒 → 到时间自动提醒创作

**Phase 3 完成标准**：
用户在WhatsApp发"帮我分析餐饮B2B机会"，AI自动生成战略报告 → 询问是否生成内容 → 用户确认 → 自动生成小红书文案 → 合规检查 → 发给用户确认发布。全程不需要打开网页。

## 五、六脉方法论文档化（核心IP资产）

> 这是独立于技术栈的核心资产，即使换掉所有技术，方法论不变。

### 文档化计划（Phase 1期间启动）

建立 `docs/METHODOLOGY.md`，内容结构：

```
# 六脉增长方法论

## 总论
- 六脉的哲学基础（天地人神财法）
- 六脉之间的关系和数据流
- 适用场景和目标用户

## 天道·战略（天）
- 定义：商业战略定位与路线图
- 输入：赛道、营收目标、创始人故事、核心优势
- 分析框架：PEST + 波特五力 + SWOT + 蓝海战略
- 输出：战略总纲（定位、差异化、路线图、风险评估）
- 质量标准：什么样的输出才算合格
- 与其他五脉的上下文传递

## 神韵·内容（神）
- 定义：品牌IP内容创作引擎
- 前置条件：需要天道·战略的输出作为上下文
- 内容类型：短视频脚本/图文/社交帖子/长文
- 平台适配：小红书/抖音/公众号/视频号
- 质量标准：原创性、传播性、转化导向

## 地势·产业（地）
- 定义：供应链与产业生态分析
- [待细化]

## 人和·流量（人）
- 定义：用户增长与流量获取
- [待细化]

## 财运·转化（财）
- 定义：商业变现与转化优化
- [待细化]

## 法度·风控（法）
- 定义：合规审查与风险防控
- [待细化]
```

### 文档化方式

你可以用以下方法逐步完善：
1. 每次完善一个模块的方法论时，同步更新文档
2. 用AI辅助：把你脑子里的想法说出来，让AI整理成结构化文档
3. 从实际用户的使用反馈中提炼和优化方法论
