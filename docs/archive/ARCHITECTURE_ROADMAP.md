## 📝 开发备忘

### 核心原则（不可妥协）

```
1. 去中心化 > 中心化
2. 异步 > 同步
3. 简单 > 复杂
4. 渐进 > 一步到位
5. 数据 > 算法
6. 用户体验 > 技术完美
```

### 禁止事项（铁律）

```
❌ 中心化所有Agent通信
❌ 同步等待Agent响应
❌ 过早抽象（先让一脉完美）
❌ 忽视匿名用户（免费用户也是用户）
❌ 硬编码Agent逻辑（需要可配置）
❌ 忽视错误处理（必须有优雅降级）
```

### 成功标准（每一Phase）

```
Phase 1: 一个不懂技术的中小企业主，能生成战略报告
Phase 2: 六脉之间可以共享上下文
Phase 3: Telegram用户能体验蜂群协同
Phase 4: AGI能主动追问并从反馈中学习
```
