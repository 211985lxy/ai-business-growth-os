## 🎯 三层关系

```

    ↓ 依赖
第三层（底层）
```

### 为什么这样分层？

**1. 天人合一**
- 天（天道、地利）= 产品灵魂
- 人（人和、神韵）= 产品智商
- 地（财帛、法度）= 产品根基

**2. 自上而下**
- 第一层定义"为什么"（战略层）
- 第二层定义"怎么做"（逻辑层）
- 第三层定义"用什么"（工程层）

**3. 相互支撑**
- 没有第一层，产品没有灵魂（只是一个工具）
- 没有第二层，产品没有智能（只是一个问答机）
- 没有第三层，产品活不下来（只是一个demo）

## 🔥 核心洞察

### 1. 天道·战略 = 产品的"北极星"

```
问题：为什么做 AI 麦肯锡？
答案：让每个创业者都有世界级的商业智能系统

问题：商业护城河是什么？
答案：数据飞轮 + 蜂群智能 + AGI雏形

问题：增长飞轮如何从0滚到1？
答案：单点工具 → 六脉贯通 → 蜂群协同 → AGI进化
```

### 2. 地利·产品 = 产品的"物理形态"

```
六个 Agent 的功能边界：
- 天道：方向在哪？（定位）
- 地利：卖什么？（产品）
- 人和：怎么连？（模式）
- 神韵：怎么说？（内容）
- 财帛：怎么收钱？（转化）
- 法度：如何不死？（风险）

三栏布局的交互逻辑：
- 左侧：任务输入
- 中间：过程可见
- 右侧：结果输出
```

### 3. 人和·协同 = 产品的"智商"

```
Queen Agent 如何调配六脉？
- 理解用户意图
- 分解为Agent任务
- 协调执行顺序
- 整合输出结果

主动追问逻辑：
- 识别信息缺失
- 主动补充追问
- 形成闭环决策

多 Agent 共识机制：
- 跨Agent记忆共享（Swarm Memory）
- Agent间通信协议（Swarm Protocol）
- 冲突解决策略（Queen仲裁）
```

### 4. 神韵·交互 = 产品的"体验"

```
极简主义原则：
- 零装饰
- 每个像素都有目的
- 功能 > 美观

流式输出：
- 实时反馈
- 思考可见
- 情绪共鸣

内容如何触动情绪与逻辑：
- 干货 + 故事 + 金句
- 数据 + 洞察 + 行动建议
```

### 5. 财帛·价值 = 产品的"成本效率"

```
技术栈选择：
- Dify（AI编排）
- 硅基流动（模型网关）
- Redis（记忆缓存）

如何省钱：
- 短期：Redis缓存减少API调用
- 中期：Dify工作流复用
- 长期：自有模型微调

数据资产积累：
- 用户反馈循环
- 知识蒸馏
- 模型持续优化
```

### 6. 法度·规范 = 产品的"安全底线"

```
红蓝对抗机制：
- 蓝军：设计防御机制
- 红军：模拟攻击
- 持续迭代

所有请求的超时控制：
- API调用：60秒超时
- 流式响应：120秒超时
- AbortSignal保证

输入验证：
- 长度检查
- 格式验证
- XSS防护

匿名限流：
- IP跟踪
- 数据库计数
- 5次/小时

优雅降级：
- Redis挂了？跳过记忆
- Dify挂了？返回友好提示
- 数据库挂了？localStorage兜底
```
