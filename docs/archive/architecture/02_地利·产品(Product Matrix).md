## 一、产品定义

### 1.1 我们卖什么？

**表面：** 一个 AI 商业咨询工具

**实质：** 一个世界级的商业智能系统

```
类比：
- 麦肯锡 = 顶级咨询顾问团队
- 六脉蜂群 = 7x24小时在线的 AI 麦肯锡

区别：
- 麦肯锡：几十万一次，服务有限
- 六脉蜂群：几百块一个月，无限使用
```

### 1.2 产品核心价值

```
传统咨询的问题：
  ❌ 价格昂贵（一次几十万）
  ❌ 服务有限（顾问时间有限）
  ❌ 知识碎片（无法串联）
  ❌ 难以持续（一次交付结束）

六脉蜂群的解法：
  ✅ 价格亲民（订阅制）
  ✅ 无限使用（AI 7x24小时）
  ✅ 系统化（六脉贯通）
  ✅ 持续陪伴（数据飞轮）
```

## 二、六脉功能边界

### 2.1 六脉终极定义（2026.03）

```
1. 天道（定位）：做不做？
   → 方向选择、战略定位、差异化分析

2. 地利（产品）：卖什么？
   → 产品定义、产品矩阵、产品护城河

3. 人和（模式）：怎么连？
   → 商业模式、生态设计、盟友与对手

4. 神韵（内容）：怎么说？
   → 品牌IP、内容生产、流量获取

5. 财帛（转化）：怎么收钱？
   → 变现路径、定价策略、转化漏斗

6. 法度（风险）：如何不死？
   → 合规审查、风险控制、法律边界
```

### 2.2 每个 Agent 的能力矩阵

#### 天道·战略 Agent

**输入：**
- 行业/赛道描述
- 创业者背景
- 资源情况
- 目标（如"从0到100万"）

**输出：**
- 值得做吗？（GO/NO-GO）
- 如何定位？
- 差异化是什么？
- 目标用户是谁？
- 战略路线图

**例子：**
```
用户问：美业培训能做吗？

天道分析：
❌ 美业培训是红海
✅ 但"创业者教育"是蓝海
✅ 你的10年经验是壁垒
✅ 定位：创业者实战导师
```

#### 地利·产品 Agent

**输入：**
- 战略定位（来自天道）
- 现有产品
- 目标营收

**输出：**
- 核心产品应该是什么？
- 产品矩阵如何设计？
- 产品护城河在哪？

**核心逻辑：** 产品足够长，就是产业

**例子：**
```
单一爆品 → 产品线 → 产业链
   ↓           ↓          ↓
实战课程   SaaS工具   供应链平台
```

#### 人和·模式 Agent

**输入：**
- 战略定位（来自天道）
- 产品定义（来自地利）
- 资源情况

**输出：**
- 谁是我的盟友？
- 谁是我的客户？
- 谁是我的对手？
- 如何建立生态？

**核心逻辑：** 商业模式 = 团结谁 + 对抗谁

**例子：**
```
盟友：培训机构（分成）、美业博主（佣金）
客户：美业从业者（转型）、小型店主（B端）
对手：泛培训课程（差异化）
```

#### 神韵·内容 Agent

**输入：**
- 战略定位（来自天道）
- 产品定义（来自地利）
- 目标用户

**输出：**
- 品牌人设是什么？
- 内容风格是什么？
- 在哪个平台发声？
- 什么内容能火？

**核心逻辑：** 内容时代，内容即流量

**例子：**
```
品牌人设："10年美业老兵，帮助100+学员成功"
内容矩阵：
  - 小红书：创业故事+避坑指南（日更）
  - 抖音：培训现场+学员访谈（周更）
  - 公众号：深度方法论（月更）
```

#### 财帛·转化 Agent

**输入：**
- 产品定义（来自地利）
- 商业模式（来自人和）
- 目标营收

**输出：**
- 变现路径怎么设计？
- 定价策略是什么？
- 转化漏斗如何优化？
- 如何最大化 LTV？

**核心逻辑：** 不是一次交易，而是长期关系

**例子：**
```
变现路径：
  免费（小红书内容）
    ↓ 3% 转化
  低价（99元体验课）
    ↓ 15% 转化
  中价（2990元系统课）
    ↓ 30% 转化
  高价（19800元私教）
```

#### 法度·风险 Agent

**输入：**
- 内容文案
- 发布平台
- 业务场景

**输出：**
- 哪些词不能用？
- 哪些承诺不能做？
- 哪些规则要注意？
- 如何避免"翻车"？

**核心逻辑：** 活下来，比做大更重要

**例子：**
```
❌ 不能说："第一"、"最"、"保证"
✅ 建议说："已帮助100+学员成功"

❌ 不能承诺："包就业"、"月入10万"
✅ 必须明确：风险提示、退款政策
```

## 三、交互逻辑

### 3.1 三栏布局（Workplace）

```
┌─────────────────────────────────────────────────────┐
│  六脉蜂群 · Workplace                                 │
├──────────────┬──────────────────┬────────────────────┤
│   左侧栏     │     中间栏       │      右侧栏        │
│  (任务输入)   │   (过程可见)     │    (结果输出)      │
│              │                  │                    │
│  选择 Agent  │   执行进度       │   流式输出         │
│  输入参数    │   当前状态       │   最终结果         │
│  开始执行    │   中间步骤       │   历史记录         │
│              │   思考可见       │   复制按钮         │
│              │                  │                    │
└──────────────┴──────────────────┴────────────────────┘
```

### 3.2 交互流程

#### 单 Agent 调用

```
1. 用户选择 Agent（如"天道·战略"）
2. 输入问题（如"美业培训能做吗？"）
3. 点击"开始执行"
4. 中间栏显示：正在分析赛道...
5. 右侧栏流式输出：分析结果
```

#### 多 Agent 协同（Queen Agent 调度）

```
1. 用户输入：我想做美业培训，从0到100万
2. Queen Agent 分解任务：
   - 天道·战略：能做吗？如何定位？
   - 地利·产品：卖什么产品？
   - 人和·模式：如何设计商业模式？
   - ...
3. 按顺序执行，每个 Agent 的结果传递给下一个
4. 最终输出：完整商业计划
```

### 3.3 状态可见

**用户需要看到什么？**

1. **当前在做什么**
   - "正在分析赛道..."
   - "正在定义产品..."
   - "正在设计商业模式..."

2. **进度如何**
   - 进度条（如果有多个步骤）
   - 预计时间（基于历史数据）

3. **中间结果**
   - 思考过程可见
   - 数据来源透明
   - 逻辑链路清晰

## 四、模块拆解

### 4.1 如何将复杂的商业问题拆解为可操作的模块？

**核心思想：** 分而治之

```
复杂问题："如何从0到100万营收？"

Queen Agent 分解：
  ↓
┌─────────────────────────────────────┐
│ 1. 天道·战略                         │
│    - 这个赛道值得做吗？              │
│    - 如何差异化定位？                │
│  - 需要输入：行业、背景、资源        │
└─────────────────────────────────────┘
  ↓
┌─────────────────────────────────────┐
│ 2. 地利·产品                         │
│    - 核心产品是什么？                │
│    - 产品矩阵如何设计？              │
│  - 需要输入：战略定位（来自天道）    │
└─────────────────────────────────────┘
  ↓
┌─────────────────────────────────────┐
│ 3. 人和·模式                         │
│    - 谁是盟友？谁是客户？            │
│    - 如何设计商业模式？              │
│  - 需要输入：战略+产品（来自前两脉） │
└─────────────────────────────────────┘
  ↓
... (继续神韵、财帛、法度)
  ↓
最终输出：完整的商业计划
```

### 4.2 模块依赖关系

```
天道（独立）
  ↓
地利（依赖天道）
  ↓
人和（依赖天道+地利）
  ↓
神韵（依赖天道+地利+人和）
  ↓
财帛（依赖人和+神韵）
  ↓
法度（依赖所有）
```

### 4.3 Swarm Memory（跨模块记忆）

**问题：** 如何让后面的 Agent 知道前面的 Agent 做了什么？

**解法：** Swarm Memory

```
天道执行完毕 → 记忆保存到 Redis
  ↓
地利执行前 → 从 Redis 读取天道的结论
  ↓
地利执行完毕 → 更新 Redis
  ↓
人和执行前 → 从 Redis 读取天道+地利的结论
...
```

**记忆结构：**

```typescript
SwarmMemory {
  business_profile: "美业培训，10年经验",      // 天道
  strategic_goal: "定位创业者实战导师",        // 天道
  product_matrix: "核心产品：实战课程",        // 地利
  market_insights: "盟友：培训机构...",        // 人和
  brand_voice: "品牌人设：10年老兵...",        // 神韵
  monetization_path: "免费→低价→中价...",      // 财帛
  risk_constraints: "禁用词：第一、最...",     // 法度
  last_pulse_summary: "[天道执行完毕]",        // 上一个Agent
}
```

## 五、产品边界

### 5.1 我们做什么？

```
✅ 商业战略分析
✅ 产品定义与设计
✅ 商业模式设计
✅ 内容生产与建议
✅ 变现路径设计
✅ 风险合规审查
```

### 5.2 我们不做什么？

```
❌ 具体执行（我们给建议，不替你做）
❌ 法律建议（合规提醒，不替代律师）
❌ 财务建议（定价策略，不替代会计师）
❌ 情绪咨询（商业问题，不处理情绪问题）
```

### 5.3 产品原则

```
1. 清晰 > 复杂
   - 功能边界清晰
   - 交互逻辑简单

2. 系统 > 碎片
   - 六脉贯通
   - 闭环输出

3. 有用 > 花哨
   - 解决实际问题
   - 不追求功能堆砌
```
