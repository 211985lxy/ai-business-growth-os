- 有产品、有服务、有团队

- 请不起麦肯锡（一次咨询几十万）
- 买了各种课程，但碎片化知识无法串联

**AI 时代的新机会：**

如果能把麦肯锡级别的商业分析能力，通过 AI 民主化到每个创业者？
如果创业者有一个 24/7 的 AI 战略顾问，随时问任何商业问题？
如果从战略到内容到执行，有一个完整系统能自动完成？

**这就是六脉增长系统的原点。**

## 二、六脉哲学（What）

### 2.1 为什么是"六脉"？

**不是随意起的名字。**

灵感来源于《天龙八部》的六脉神剑：
- 六条经脉，六种剑气
- 相生相克，形成完整武学体系
- 单脉强劲，六脉合一则天下无敌

**对应到商业：**

商业也是一个复杂的系统，需要六个维度协同：

| 脉 | 中国哲学 | 商业维度 | 简单理解 |
|----|---------|---------|---------|
| 天道·战略 | 天 | 方向、定位、愿景 | 去哪里？ |
| 神韵·内容 | 神 | 品牌、IP、文化 | 谁是我？ |
| 地道·产业 | 地 | 供应链、资源 | 有什么？ |
| 人道·流量 | 人 | 用户、渠道、增长 | 找谁买？ |
| 财帛·转化 | 财 | 变现、ROI、盈利 | 怎么赚？ |
| 法度·风险 | 法 | 合规、风控、底线 | 不能做什么？ |

**这六个维度，构成了一个完整的商业系统。**

缺任何一"脉"，商业都不完整：
- 只有战略没有内容 = 纸上谈兵
- 只有内容没有流量 = 无人知晓
- 只有流量没有转化 = 空热闹
- 只有转化没有风控 = 随时崩盘

---

### 2.2 六脉之间的关系

```
        天道·战略（顶层设计）
              ↓
    ┌─────────┴─────────┐
    ↓                   ↓
神韵·内容          地道·产业
(品牌表达)          (资源支撑)
    ↓                   ↓
    → 人道·流量 ← → 财帛·转化 ←
            (获取与变现)
              ↓
          法度·风险
          (安全边界)
```

**数据流逻辑：**

1. **天道·战略**先定方向（定位、差异化、路线图）
2. **神韵·内容**根据战略生成品牌表达（文案、脚本、视觉）
3. **地道·产业**提供资源支撑（供应链、合作伙伴、行业数据）
4. **人道·流量**找到目标用户（渠道、获客、用户画像）
5. **财帛·转化**设计变现路径（定价、成交话术、私域运营）
6. **法度·风险**全程合规检查（广告法、平台规则、法律风险）

**每一脉的输出，都是下一脉的输入。**

这就是"六脉贯通"。

---

### 2.3 单脉也能独立价值

虽然六脉形成闭环，但**每一脉都可以独立解决问题**：

- 只有战略困惑？用天道·战略
- 只缺内容能力？用神韵·内容
- 只想找流量渠道？用人道·流量
- 只需风控审核？用法度·风险

**渐进式采用路径：**

```
免费体验天道·战略（获客入口）
         ↓
    感受到价值
         ↓
付费解锁六脉全通（完整闭环）
```

## 三、设计哲学（How）

### 3.1 设计标杆

我们不是在模仿谁，而是在吸收最好的实践：

| 产品 | 我们吸收的是什么 |
|------|----------------|
| Google AI Studio | 三栏布局的简洁高效 |
| 飞书 | 暗色模式的专业感 |
| Linear | 极致的简洁，零花哨 |
| ChatGPT | 流式输出的体验 |
| 麦肯锡报告 | 战略分析框架 |

**核心理念：**

```
干净 > 花哨
功能 > 美观
一致 > 个性
```

---

### 3.2 UI 设计哲学

**为什么这么干净？**

因为我们的用户是创业者，他们的注意力很宝贵。

- 每个像素都必须有功能
- 没有装饰性的 SVG 曲线
- 没有花哨的动画
- 没有emoji 作为 UI 元素

**为什么是三栏布局？**

```
[历史] | [输入] | [输出]
  ↑       ↑        ↑
过去    现在     未来
```

这是人类认知的逻辑：
- 左边看过去（历史记录）
- 中间操作现在（输入参数）
- 右边看见未来（AI 生成结果）

**为什么要有折叠按钮？**

因为用户需要专注：
- 思考时，折叠历史，只留输入
- 阅读报告时，折叠输入，只留输出
- 这种"可伸缩的工作空间"，是 AI 时代的界面范式

---

### 3.3 交互设计哲学

**流式输出的意义：**

不是技术炫技，而是用户体验的必需。

为什么？
- AI 生成需要时间（30秒 - 3分钟）
- 如果黑屏等待，用户会焦虑
- 流式输出 = 看见 AI 在思考
- 这建立了信任感

**暂停按钮的意义：**

用户需要控制。
- 想停下来调整参数？可以
- 想早点看结果，不继续生成？可以
- 把控制权还给用户

**历史记录的意义：**

AI 不是一次性工具，是长期伙伴。
- 上周问的问题，今天还能查看
- 上月的策略，今天还能优化
- 知识的积累，比单次生成更重要

## 四、技术架构哲学

### 4.1 为什么是"智能体矩阵"？

不是一个大模型解决所有问题，而是**专才分工**：

```
战略模块 → 推理模型（DeepSeek-V3 / GLM-4）
内容模块 → 创作模型（GLM-4 / Qwen）
风控模块 → 规则模型（Qwen-7B 免费版）
```

**为什么？**

- 不同的任务需要不同的能力
- 成本优化（不需要所有任务都用最贵模型）
- 专业性（专才 > 通才）

---

### 4.2 上下文注入机制

这是六脉贯通的技术实现：

```typescript
// 天道·战略生成后
{
  niche: "美业培训",
  target_audience: "二三线城市女性创业者",
  differentiation: "标准化服务流程"
}

// 自动注入到神韵·内容
const contentPrompt = `
  基于以下战略：
  - 赛道：${niche}
  - 目标用户：${target_audience}
  - 差异化：${differentiation}

  生成小红书文案...
`;
```

**六脉之间不是孤岛，而是通过上下文连接。**

---

### 4.3 为什么是渐进式架构？

Phase 1 → Phase 2 → Phase 3 → Phase 4

```
单一模块 → 六脉贯通 → 多渠道 → 商业化
```

**为什么不是一次做全？**

1. **资源有限**：一个人/小团队，不可能同时做完美
2. **验证假设**：先让一个模块完美可用，验证市场
3. **积累数据**：用户越多，行业数据越丰富，后续模块越精准
4. **降低风险**：如果战略模块没人用，就不必投入做其他模块

**"先让一脉完美，再贯通六脉"**

---

### 4.4 数据持久化策略

```sql
strategy_contexts  -- 战略上下文（跨模块共享）
agent_outputs      -- 所有智能体的输出日志
user_profiles      -- 用户画像（积累的行业数据）
```

**为什么重要？**

- AI 不是一次性工具，是长期伙伴
- 用户的商业知识需要积累
- 数据越多，AI 越懂用户
- 这形成护城河

## 五、与竞品的本质区别

| 竞品类型 | 他们的逻辑 | 我们的逻辑 |
|---------|-----------|-----------|
| ChatGPT | 问什么答什么 | 主动引导，系统化分析 |
| 咨询公司 | 人工服务，昂贵 | AI 民主化，低成本 |
| SaaS 工具 | 单点功能（如文案生成） | 六脉闭环 |
| 课程培训 | 知识碎片化 | 方法论系统化 |

**我们的护城河：**

不是技术（技术都能学），而是：
1. **六脉方法论**（这是 IP）
2. **数据飞轮**（用户越多，越精准）
3. **OpenClaw 自动化**（从"生成"到"执行"）

## 六、最终愿景（Where）

### 6.1 短期愿景（1年内）

```
一个完全不懂技术的中小企业主，
打开网站，输入"我做美业培训"，
3分钟内得到一份高质量战略报告。
```

**衡量标准：**
- 零报错、零困惑
- 输出质量达到麦肯锡初级顾问水平
- 用户愿意为此付费

---

### 6.2 中期愿景（2年内）

```
用户在 WhatsApp 发一条消息：
"帮我分析餐饮 B2B 机会"

系统自动：
1. 生成战略报告
2. 询问是否生成内容
3. 用户确认 → 生成小红书文案
4. 合规检查
5. 询问是否发布
6. 自动发布到多个平台

全程不需要打开网页。
```

**从"打开网页生成报告"到"发条消息自动执行"**

---

### 6.3 长期愿景（5年内）

```
每个创业者都有一个 AI 商业合伙人。

不是工具，是伙伴。
不是回答，是共同决策。
不是执行，是一起成长。
```

**六脉增长系统 = 创业者的 external brain**
