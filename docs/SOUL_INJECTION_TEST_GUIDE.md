### ✅ Phase 3: API 测试

#### 3.1 文件上传 API 测试
使用 curl 或 Postman 测试：
```bash
curl -X POST http://localhost:3000/api/knowledge/upload \
  -F "file=@test.pdf" \
  -F "meridian_type=shen"
```

预期响应：
```json
{
  "success": true,
  "fileId": "xxx",
  "syncStatus": "indexing",
  "difyFileId": "yyy"
}
```

测试要点：
- [ ] 成功上传文件
- [ ] 返回正确的文件 ID
- [ ] 返回同步状态
- [ ] 返回 Dify 文件 ID
- [ ] 测试不支持的文件格式
- [ ] 测试空文件

#### 3.2 同步状态查询 API 测试
使用浏览器访问：
```
http://localhost:3000/api/knowledge/sync-status?user_id=YOUR_USER_ID
```

预期响应：
```json
{
  "totalFiles": 42,
  "indexedFiles": 40,
  "failedFiles": 2,
  "coveragePercentage": 95,
  "syncStatus": "synced",
  "lastSyncAt": "2024-01-01T00:00:00.000Z",
  "meridianStats": {
    "tian": { "count": 8, "wordCount": 15000 },
    "di": { "count": 6, "wordCount": 12000 },
    ...
  },
  "totalWordCount": 84000
}
```

测试要点：
- [ ] 返回正确的文件统计
- [ ] 返回覆盖率百分比
- [ ] 返回同步状态
- [ ] 返回脉络统计
- [ ] 测试不存在的用户 ID

## 🔧 快速修复指南

### 常见问题 1: 页面空白
**可能原因**: 组件导入错误
**解决方案**:
```bash
npm install mermaid
```

### 常见问题 2: API 调用失败
**可能原因**: 环境变量未配置
**解决方案**:
检查 `.env.local` 文件，确保配置了：
- `NEXT_PUBLIC_SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `DIFY_API_URL`
- `DIFY_API_KEY`

### 常见问题 3: 样式问题
**可能原因**: Tailwind CSS 未正确加载
**解决方案**:
```bash
npm run build
npm run dev
```
