## 一、六脉终极定义

```
1. 天道（定位）：做不做？
2. 地利（产品）：卖什么？
3. 人和（模式）：怎么连？
4. 神韵（内容）：怎么说？
5. 财帛（转化）：怎么收钱？
6. 法度（风险）：如何不死？
```

### 核心逻辑

```
战略 → 产品 → 模式 → 内容 → 变现 → 合规
（方向）（卖什么）（怎么团结）（获客）（赚钱）（边界）
```

## 二、详细解读

### 2.1 天道·战略：做不做？

**核心问题：这个生意值得做吗？**

```
创业者最怕的：选错赛道，一年白干

天道·战略 Agent 回答：
- 市场规模有多大？
- 竞争是否红海？
- 我有没有优势？
- 赚钱的概率大吗？

输出：GO / NO-GO 决策 + 如果做，怎么定位
```

**对用户的话术：**
```
"别盲目开始！先让我帮你分析：
- 这个赛道值不值得做？
- 你的优势能变现吗？
- 成功的概率有多大？
- 如果做，怎么差异化定位？"
```

**例子：**
```
用户问："美业培训能做吗？"

天道分析：
❌ 美业培训是红海
✅ 但"创业者教育"是蓝海
✅ 你的10年经验是壁垒
✅ 结论：可以做，定位"创业者实战导师"
```

**输出内容：**
- 值得做吗？（GO/NO-GO）
- 如何定位？
- 差异化是什么？
- 目标用户是谁？
- 战略路线图

### 2.2 地利·产品：卖什么？

**核心问题：我卖什么产品/服务？**

```
选对赛道后，接下来定义产品

地利·产品 Agent 回答：
- 核心产品是什么？（单一爆品）
- 产品矩阵怎么设计？（如何扩展）
- 产品护城河在哪？（如何形成壁垒）

核心逻辑：产品足够长，就是产业
```

**对用户的话术：**
```
"选对赛道了，接下来：
- 核心产品应该是什么？
- 不要想着做全产业链
- 先做一个爆品，再扩展成产品矩阵
- 产品足够长，自然形成产业"
```

**例子：**
```
用户问："我的核心产品应该是什么？"

地利分析：
❌ 不要一开始就想做全产业链
✅ 先做一个"爆品"：美业创业者实战课
✅ 第二步：延伸到"工具化"：培训 SaaS
✅ 第三步：扩展到"生态化"：供应链平台

产品进化路径：单一爆品 → 产品线 → 产业链
```

**输出内容：**
- 核心产品定义
- 产品矩阵设计
- 产品护城河

### 2.3 人和·模式：怎么连？

**核心问题：如何设计生态位？**

```
有产品了，但一个人打不了天下

人和·模式 Agent 回答：
- 谁是我的盟友？（上游供应商、下游渠道）
- 谁是我的客户？（C端、B端、政府）
- 谁是我的对手？（直接竞争、间接竞争）
- 如何团结更多人？（建立生态）

核心逻辑：商业模式 = 团结谁 + 对抗谁
```

**对用户的话术：**
```
"有产品了，但一个人打不了天下：
- 谁能成为你的盟友？
- 谁是你的精准客户？
- 怎么建立生态？
- 不是单打独斗，是协同作战"
```

**例子：**
```
用户问："美业培训如何获客？"

人和分析：
❌ 不要只想着投放广告（成本高、效果差）
✅ 盟友：培训机构（分成：30%-70%）
✅ 盟友：企业内训（B端大客户）
✅ 客户：想转行的美业从业者（精准需求）
✅ 生态：打造"美业创业者联盟"

核心：不是单打独斗，而是生态协同
```

**输出内容：**
- 盟友识别
- 客户细分
- 竞争对手分析
- 生态设计

### 2.4 神韵·内容：怎么说？

**核心问题：如何占领用户认知？**

```
有产品、有模式，但用户不知道你

神韵·内容 Agent 回答：
- 你的品牌人设是什么？（专家/朋友/导师？）
- 你的内容风格是什么？（专业/亲和/搞笑？）
- 在哪个平台发声？（小红书/抖音/公众号）
- 什么内容能火？（痛点/干货/故事？）

核心逻辑：内容时代，内容即流量
```

**对用户的话术：**
```
"酒香也怕巷子深：
- 怎么让别人知道你？
- 不是硬广，是内容
- 内容即流量，流量即增长
- 好内容 = 免费获客"
```

**例子：**
```
用户问："如何让更多人知道我的美业培训？"

神韵分析：
❌ 不要硬广（平台会限流）
✅ 人设："10年美业老兵，帮助100+创业者成功"
✅ 平台：小红书（图文）+ 抖音（视频）
✅ 内容：
  - 干货：美业避坑指南
  - 故事：学员转型案例
  - 金句："美业不是技术生意，是人心生意"
✅ 策略：日更（3-5条）→ 积累粉丝 → 转化变现

核心：内容 = 免费、精准、持久的流量
```

**输出内容：**
- 品牌人设
- 内容矩阵
- 平台策略
- 内容示例

### 2.5 财帛·转化：怎么收钱？

**核心问题：如何设计变现路径？**

```
有流量了，但流量不等于钱

财帛·转化 Agent 回答：
- 变现路径怎么设计？（免费→低价→中价→高价）
- 定价策略是什么？（成本/价值/竞争）
- 转化漏斗如何优化？（认知→兴趣→付费→复购）
- LTV（客户终身价值）如何最大化？

核心逻辑：不是一次交易，而是长期关系
```

**对用户的话术：**
```
"有流量了，但流量不等于钱：
- 不要只想着高价
- 设计变现路径（免费→低价→中价→高价）
- 降低门槛，提高复购
- 客户终身价值 > 单次交易"
```

**例子：**
```
用户问："美业培训怎么收费？"

财帛分析：
❌ 不要只卖高价课（转化率低）
✅ 设计变现路径：

  免费层：小红书干货内容（获客）
    ↓ 3% 转化
  低价层：99元体验课（尝鲜）
    ↓ 15% 转化
  中价层：2990元系统课（主力）
    ↓ 30% 转化
  高价层：19800元私教（高客单）

核心：降低门槛，提高复购，最大化 LTV
```

**输出内容：**
- 变现路径设计
- 定价策略
- 转化漏斗
- LTV 策略

### 2.6 法度·风险：如何不死？

**核心问题：如何避免"翻车"？**

```
生意做大了，但风险也在增加

法度·风险 Agent 回答：
- 哪些词不能用？（广告法违禁词）
- 哪些承诺不能做？（法律风险）
- 哪些平台规则要注意？（封号风险）
- 哪些钱不能赚？（道德底线）

核心逻辑：活下来，比做大更重要
```

**对用户的话术：**
```
"做大了，风险也在增加：
- 哪些话不能说？
- 哪些钱不能赚？
- 如何避免翻车？
- 活下来，比做大更重要"
```

**例子：**
```
用户问："美业培训如何避免风险？"

法度分析：
❌ 不能承诺："包就业"、"月入10万"（虚假宣传）
❌ 不能说："行业第一"、"最专业"（绝对化用语）
✅ 建议说："已帮助100+学员成功转型"（可验证）
✅ 必须避坑：
  - 广告法：不用"第一"、"最"、"顶级"
  - 平台规则：不引导私聊、不直接留微信
  - 合同风险：明确退款政策、不夸大效果

核心：合规不是束缚，是护城河
```

**输出内容：**
- 风险清单
- 合规建议
- 平台规则
- 道德底线

## 三、功能边界

### 3.1 我们做什么？

```
✅ 商业战略分析
✅ 产品定义与设计
✅ 商业模式设计
✅ 内容生产与建议
✅ 变现路径设计
✅ 风险合规审查
```

### 3.2 我们不做什么？

```
❌ 具体执行（我们给建议，不替你做）
❌ 法律建议（合规提醒，不替代律师）
❌ 财务建议（定价策略，不替代会计师）
❌ 情绪咨询（商业问题，不处理情绪问题）
```

### 3.3 产品原则

```
1. 清晰 > 复杂
   - 功能边界清晰
   - 交互逻辑简单

2. 系统 > 碎片
   - 六脉贯通
   - 闭环输出

3. 有用 > 花哨
   - 解决实际问题
   - 不追求功能堆砌
```

## 四、实战案例

### 案例：美业培训创业者

```
创业者：王姐，10年美业经验，想做培训生意

[天道·战略] 做不做？
分析：美业培训是红海，但"创业者教育"是蓝海
      你的10年经验是壁垒
      结论：可以做，定位"创业者实战导师"

[地利·产品] 卖什么？
核心产品：《美业创业实战课》（12周线上+线下）
产品矩阵：
  - 单一爆品：实战课（2990元）
  - 工具化：学员 SaaS 系统（99元/月）
  - 生态化：美业供应链平台（未来）

[人和·模式] 怎么连？
盟友：
  - 合作培训机构（提供场地，分成30-70%）
  - 美业博主（推荐学员，佣金20%）
客户：
  - 想转行的美业从业者（精准）
  - 小型美业店主（B端）
对手：
  - 泛培训课程（差异化：实战 vs 理论）
  - 在线学习平台（差异化：陪伴 vs 一次性）

[神韵·内容] 怎么说？
品牌人设："10年美业老兵，帮助100+学员成功"
内容矩阵：
  - 小红书：创业故事+避坑指南（日更）
  - 抖音：培训现场+学员访谈（周更）
  - 公众号：深度方法论（月更）
流量策略：
  - 自然流量：SEO + 平台推荐
  - 社交裂变：学员口碑传播
  - 付费放大：occasional 投放（ROI > 3）

[财帛·转化] 怎么收钱？
变现路径：
  - 免费：小红书内容（获客）
  - 引流：99元直播课（转化率3%）
  - 主力：2990元系统课（转化率15%）
  - 高价：19800元私教（转化率5%）
  - 持续：99元/月 SaaS工具（复购率40%）
  - B端：企业内训（客单价5万+）

[法度·风险] 如何不死？
合规清单：
  - 广告法：不用"第一"、"最"、"保证"
  - 平台规则：不引导私聊、不直接留微信
  - 法律风险：明确退款政策、不夸大效果
  - 道德底线：不收割小白、不卖焦虑

数据飞轮：
  用户反馈 → 持续优化课程 → 口碑更好 → 更多用户
```

## 五、产品形态

### 5.1 三栏布局（Workplace）

```
┌─────────────────────────────────────────────────────┐
│  六脉蜂群 · Workplace                                 │
├──────────────┬──────────────────┬────────────────────┤
│   左侧栏     │     中间栏       │      右侧栏        │
│  (任务输入)   │   (过程可见)     │    (结果输出)      │
│              │                  │                    │
│  选择 Agent  │   执行进度       │   流式输出         │
│  输入参数    │   当前状态       │   最终结果         │
│  开始执行    │   中间步骤       │   历史记录         │
│              │   思考可见       │   复制按钮         │
│              │                  │                    │
└──────────────┴──────────────────┴────────────────────┘
```

### 5.2 交互流程

**单 Agent 调用：**
```
1. 用户选择 Agent（如"天道·战略"）
2. 输入问题（如"美业培训能做吗？"）
3. 点击"开始执行"
4. 中间栏显示：正在分析赛道...
5. 右侧栏流式输出：分析结果
```

**多 Agent 协同（Queen Agent 调度）：**
```
1. 用户输入：我想做美业培训，从0到100万
2. Queen Agent 分解任务
3. 按顺序执行，每个 Agent 的结果传递给下一个
4. 最终输出：完整商业计划
```

### 5.3 状态可见

**用户需要看到什么？**
1. **当前在做什么** - "正在分析赛道..."
2. **进度如何** - 进度条、预计时间
3. **中间结果** - 思考过程可见

## 六、数据流

### 6.1 依赖关系

```
天道（独立）
  ↓
地利（依赖天道）
  ↓
人和（依赖天道+地利）
  ↓
神韵（依赖天道+地利+人和）
  ↓
财帛（依赖人和+神韵）
  ↓
法度（依赖所有）
```

### 6.2 Swarm Memory（跨Agent记忆）

```
天道执行完毕 → 记忆保存到 Redis
  ↓
地利执行前 → 从 Redis 读取天道的结论
  ↓
地利执行完毕 → 更新 Redis
  ↓
人和执行前 → 从 Redis 读取天道+地利的结论
...
```
