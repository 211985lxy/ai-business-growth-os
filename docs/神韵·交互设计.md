## 一、设计哲学

### 1.1 核心原则

```
1. Clean > Fancy
   清晰 > 花哨

2. Functional > Beautiful
   功能 > 美观

3. Consistent > Unique
   一致 > 独特

4. Simple > Complex
   简单 > 复杂
```

### 1.2 设计目标

```
不是让用户欣赏界面
而是让用户专注于内容

每个像素都有目的
不要纯装饰元素
```

### 1.3 为什么极简？

**认知负荷理论：**
```
人类工作记忆容量：7 ± 2 个元素

如果界面有太多元素：
  ❌ 用户需要花时间理解"怎么用"
  ❌ 分散注意力，找不到核心功能
  ❌ 产生焦虑和挫败感

极简设计的好处：
  ✅ 用户一眼就知道"做什么"
  ✅ 专注于内容，而不是界面
  ✅ 快速、流畅、高效
```

## 二、视觉规范

### 2.1 颜色系统

**主色调（功能性颜色）：**

```
黑/灰系 - 内容
├── text-primary:     #0f172a  (slate-900)  主要内容
├── text-secondary:   #475569  (slate-600)  次要内容
├── text-muted:       #94a3b8  (slate-400)  弱化内容
└── bg-primary:       #ffffff  (white)      主背景

蓝系 - 可交互
├── primary:          #4f46e5  (indigo-600) 主交互
├── primary-hover:    #4338ca  (indigo-700) 悬停状态
└── primary-light:    #e0e7ff  (indigo-100) 浅背景

状态色
├── success:          #22c55e  (green-500)  成功
├── warning:          #eab308  (yellow-500) 警告
├── error:            #ef4444  (red-500)    错误
└── info:             #3b82f6  (blue-500)   信息
```

**使用原则：**
```
✅ 只用功能性颜色
   - 黑色/灰色：内容
   - 蓝色：可交互
   - 绿/黄/红：状态

❌ 不使用装饰性颜色
   - 渐变色（除非有功能性）
   - 花纹、图案
   - 不必要的阴影
```

### 2.2 字体系统

**字体家族：**
```
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI",
             "Noto Sans", "Helvetica Neue", Arial,
             "Noto Sans SC", "PingFang SC", "Microsoft YaHei",
             sans-serif;
```

**字体大小：**
```
├── xs:   0.75rem   (12px)  辅助信息
├── sm:   0.875rem  (14px)  次要内容
├── base: 1rem      (16px)  正文（默认）
├── lg:   1.125rem  (18px)  小标题
├── xl:   1.25rem   (20px)  标题
└── 2xl:  1.5rem    (24px)  大标题
```

**字体粗细：**
```
├── normal:   400  正文
├── medium:   500  强调
└── semibold: 600  标题
```

### 2.3 间距系统

**间距单位（0.25rem = 4px）：**
```
├── 1:  0.25rem  (4px)   最小间距
├── 2:  0.5rem   (8px)   紧凑间距
├── 3:  0.75rem  (12px)  小间距
├── 4:  1rem     (16px)  标准间距
├── 6:  1.5rem   (24px)  中等间距
├── 8:  2rem     (32px)  大间距
└── 12: 3rem     (48px)  超大间距
```

**使用原则：**
```
- 用 4 的倍数（4px, 8px, 16px, 24px...）
- 保持一致的节奏感
- 不要用奇怪的间距（如 7px, 13px）
```

### 2.4 圆角

```
├── sm:  0.25rem  (4px)   小圆角（按钮、标签）
├── md:  0.375rem (6px)   中圆角（卡片）
├── lg:  0.5rem   (8px)   大圆角（容器）
└── full: 9999px          完全圆角（头像、徽章）
```

**使用原则：**
```
✅ 圆角表示"可点击"
✅ 统一使用相同的圆角大小
❌ 不要混用太多不同的圆角
```

## 三、组件设计

### 3.1 按钮（Button）

**基础样式：**
```tsx
// 主按钮
<button className="bg-indigo-600 hover:bg-indigo-700 text-white px-4 py-2 rounded-md">
  确定
</button>

// 次按钮
<button className="bg-white border border-gray-300 hover:bg-gray-50 text-gray-700 px-4 py-2 rounded-md">
  取消
</button>

// 文本按钮
<button className="text-indigo-600 hover:text-indigo-700 px-2 py-1">
  了解更多
</button>
```

**尺寸规范：**
```
├── sm:  h-8 px-3 text-sm   小按钮
├── md:  h-10 px-4 text-base  标准按钮（默认）
└── lg:  h-12 px-6 text-lg   大按钮
```

**状态：**
```
├── 默认：正常显示
├── 悬停：hover:bg-indigo-700
├── 禁用：opacity-50 cursor-not-allowed
└── 加载：spinner + 禁用状态
```

### 3.2 输入框（Input）

**基础样式：**
```tsx
<input
  type="text"
  className="w-full px-3 py-2 border border-gray-300 rounded-md
             focus:outline-none focus:ring-2 focus:ring-indigo-500
             focus:border-transparent"
  placeholder="请输入..."
/>
```

**状态：**
```
├── 默认：border-gray-300
├── 聚焦：ring-2 ring-indigo-500
├── 错误：border-red-500 ring-red-500
└── 禁用：bg-gray-100 cursor-not-allowed
```

### 3.3 卡片（Card）

**基础样式：**
```tsx
<div className="bg-white border border-gray-200 rounded-lg p-6">
  <h3 className="text-lg font-semibold mb-2">标题</h3>
  <p className="text-gray-600">内容</p>
</div>
```

**变体：**
```
├── 默认卡片：白色背景 + 边框
├── 强调卡片：indigo-50 背景
└── 无边框卡片：只有背景色
```

### 3.4 折叠面板（Collapse）

**Workplace 三栏布局的折叠按钮：**

```tsx
// 左侧栏折叠按钮
<button className="w-6 h-12 flex items-center justify-center
                      hover:bg-indigo-50 text-gray-500 hover:text-indigo-600
                      rounded-r-md border-y border-r border-gray-200">
  <ChevronLeft className="w-4 h-4" />
</button>

// 右侧栏折叠按钮
<button className="w-6 h-12 flex items-center justify-center
                      hover:bg-indigo-50 text-gray-500 hover:text-indigo-600
                      rounded-l-md border-y border-l border-gray-200">
  <ChevronRight className="w-4 h-4" />
</button>
```

**规范：**
```
├── 尺寸：w-6 h-12（不是 w-7 h-14）
├── 悬停：hover:bg-indigo-50（不是 dark mode）
├── 图标：w-4 h-4（不是 w-5 h-5）
└── 颜色：gray-500 → indigo-600（不是 slate-700）
```

### 3.5 徽章（Badge）

**基础样式：**
```tsx
<Badge className="bg-indigo-100 text-indigo-700">
  新功能
</Badge>
```

**颜色：**
```
├── 默认：indigo
├── 成功：green
├── 警告：yellow
└── 错误：red
```

### 3.6 加载（Loading）

**基础样式：**

```
├── 小：w-4 h-4
├── 中：w-8 h-8（默认）
└── 大：w-12 h-12
```

## 四、交互模式

### 4.1 流式输出

**为什么流式？**
```
传统方式（等待后输出）：
  用户提交 → 等待10秒 → 看到完整结果
  ❌ 焦虑："是不是卡住了？"
  ❌ 无聊：盯着加载动画

流式方式（边生成边输出）：
  用户提交 → 立即看到输出 → 逐字显现
  ✅ 安心："系统在思考"
  ✅ 参与：看到思考过程
```

**实现方式：**
```tsx
// React 流式输出
function StreamingOutput({ stream }) {
  const [text, setText] = useState("");

  useEffect(() => {
    const reader = stream.getReader();

    async function read() {
      while (true) {
        const { done, value } = await reader.read();
        if (done) break;
        setText(prev => prev + decoder.decode(value));
      }
    }

    read();
  }, [stream]);

  return <div className="prose max-w-none">{text}</div>;
}
```

### 4.2 即时反馈

**原则：** 用户操作 → 立即看到结果

```
✅ 好的反馈：
  点击按钮 → 按钮变色 → 显示"正在处理..."

❌ 坏的反馈：
  点击按钮 → 没反应 → 用户困惑："是不是没点到？"
```

### 4.3 错误处理

**原则：** 友好提示，不暴露技术细节

```typescript
// ❌ 技术错误
throw new Error("TimeoutError: Request timeout after 60000ms");

// ✅ 人性化错误
throw new Error("服务响应时间过长，请稍后再试。");
```

### 4.4 空状态（Empty State）

**设计：**
```tsx
<div className="text-center py-12">
  <FileText className="w-16 h-16 mx-auto text-gray-400 mb-4" />
  <h3 className="text-lg font-semibold text-gray-900 mb-2">
    还没有生成内容
  </h3>
  <p className="text-gray-500 mb-6">
    在左侧输入你的问题，开始使用六脉蜂群
  </p>
  <Button onClick={onStart}>
    开始使用
  </Button>
</div>
```

## 五、响应式设计

### 5.1 断点系统

```
├── sm:  640px   移动设备
├── md:  768px   平板设备
├── lg:  1024px  桌面设备
└── xl:  1280px  大屏幕
```

### 5.2 Workplace 响应式布局

**桌面（lg及以上）：**
```
┌─────────────────────────────────────────────────────┐
│  左侧栏(280px)  │  中间栏(自适应)  │  右侧栏(420px)  │
│  可折叠         │  最小宽度(320px)  │  可折叠         │
└─────────────────────────────────────────────────────┘
```

**平板（md）：**
```
┌─────────────────────────────────────────┐
│  左侧栏(240px)  │  中间栏 + 右侧栏      │
│  可折叠         │  垂直排列             │
└─────────────────────────────────────────┘
```

**移动（<md）：**
```
┌─────────────────────┐
│  中间栏（全屏）      │
│  左右栏抽屉式       │
└─────────────────────┘
```

### 5.3 响应式实现

```tsx
// Workplace 布局
<div className="flex h-screen">
  {/* 左侧栏 - 移动端隐藏，桌面端显示 */}
  <aside className={cn(
    "hidden md:flex md:flex-col",
    "w-[280px] lg:w-[320px]"
  )}>
    {/* ... */}
  </aside>

  {/* 中间栏 - 始终显示 */}
  <main className="flex-1 min-w-[320px]">
    {/* ... */}
  </main>

  {/* 右侧栏 - 移动端隐藏，桌面端显示 */}
  <aside className={cn(
    "hidden lg:flex lg:flex-col",
    "w-[380px] xl:w-[420px]"
  )}>
    {/* ... */}
  </aside>
</div>
```

## 六、动效设计

### 6.1 动效原则

```
✅ 功能性动效（推荐）
   - 加载动画：表示"正在处理"
   - 悬停效果：表示"可点击"
   - 过渡动画：表示"状态变化"

❌ 装饰性动效（避免）
   - 页面加载的"入场动画"
   - 元素之间的"过渡动画"
   - 鼠标移动的"跟随动画"
```

### 6.2 过渡时长

```
├── 快：150ms   按钮悬停、输入框聚焦
├── 中：300ms   折叠面板、模态框（默认）
└── 慢：500ms   复杂动画（慎用）
```

### 6.3 缓动函数

```
├── ease-out：   适合进入动画
├── ease-in：    适合离开动画
└── ease-in-out：适合往返动画（默认）
```

## 七、可访问性

### 7.1 键盘导航

```
✅ 所有交互元素支持键盘操作
✅ Tab 键顺序合理
✅ Enter/Space 触发按钮
✅ Esc 关闭模态框
```

### 7.2 屏幕阅读器

```
✅ 图片有 alt 属性
✅ 表单元素有 label
✅ 语义化 HTML（nav, main, aside）
✅ ARIA 属性（如：aria-label）
```

### 7.3 颜色对比

```
✅ 文字与背景对比度 ≥ 4.5:1（WCAG AA）
✅ 大文字对比度 ≥ 3:1
✅ 不用颜色作为唯一区分方式
```
