## 一、开发原则

### 1.1 核心心法（一页纸）

```
┌─────────────────────────────────────────┐
│          六脉蜂群 · 开发心法             │
└─────────────────────────────────────────┘

1. 产品灵魂
   ├─ 使命：让每个创业者都有世界级的商业智能系统
   ├─ 定位：AI 麦肯锡，不是工具，是合伙人
   └─ 愿景：从工具到智能，从回答到决策，从生成到执行

2. 六脉定义（v3.0 最终版）
   ├─ 天道：做不做？（战略）
   ├─ 地利：卖什么？（产品）
   ├─ 人和：怎么连？（模式）
   ├─ 神韵：怎么说？（内容）
   ├─ 财帛：怎么收钱？（转化）
   └─ 法度：如何不死？（风险）

3. 蜂群拓扑
           Queen Agent
               │
      ┌────────┼────────┐
      ↓        ↓        ↓
   天道·战略  神韵·内容  地利·产品
      ↓        ↓        ↓
   人和·模式  财帛·转化  法度·风险

4. 设计原则
   ├─ Clean > Fancy（清晰 > 花哨）
   ├─ Functional > Beautiful（功能 > 美观）
   ├─ Consistent > Unique（一致 > 独特）
   └─ Simple > Complex（简单 > 复杂）

5. 技术栈
   ├─ 前端：Next.js 16 + React 19 + TypeScript + Tailwind
   ├─ 后端：Next.js API Routes + Node.js 20.x
   ├─ 数据：Supabase (PostgreSQL) + Redis
   ├─ AI：Dify + 硅基流动 (DeepSeek-R1)
   └─ 部署：Vercel + Supabase Cloud + 自托管 Dify

6. 开发原则
   ├─ YAGNI（You Aren't Gonna Need It）
   ├─ KISS（Keep It Simple, Stupid）
   ├─ DRY（Don't Repeat Yourself）
   └─ 测试驱动（先测试，后编码）

7. 安全底线
   ├─ 所有 API 调用必须有超时控制
   ├─ 所有输入必须验证
   ├─ 匿名用户必须有速率限制
   ├─ 外部服务失败必须优雅降级
   └─ 敏感信息不能泄露到客户端

8. 用户体验
   ├─ 极简主义：零装饰
   ├─ 流式输出：实时反馈
   ├─ 错误提示：友好，不暴露技术细节
   └─ 加载状态：清晰可见

记住：我们做的是产品，不是代码。
代码是手段，产品价值是目的。
```

### 1.2 编码规范

**文件命名：**
```
✅ 好的命名：
  - user-menu.tsx（组件）
  - use-auth.ts（Hook）
  - memory-types.ts（类型）
  - API_KEY（常量）

❌ 坏的命名：
  - UserMenu.tsx（不用 Pascal Case for 文件名）
  - useAuth.ts（不用 Pascal Case for 文件名）
  - memoryTypes.ts（不用 camelCase for 文件名）
```

**变量命名：**
```
✅ 好的命名：
  - const userId = "user-123"
  - const isActive = true
  - const MAX_RETRIES = 3

❌ 坏的命名：
  - const id = "user-123"（不明确）
  - const flag = true（无意义）
  - const num = 3（不明确）
```

**函数命名：**
```
✅ 好的命名：
  - getUserById()
  - isValidEmail()
  - calculateTotal()

❌ 坏的命名：
  - user()（不明确）
  - check()（太泛）
  - process()（太泛）
```

### 2.2 API Route 模板

```typescript
/**
 * API 路由说明
 *
 * @route POST /api/example
 * @description 描述这个 API 的作用
 */

import { NextRequest, NextResponse } from "next/server";
import { createClient } from "@/lib/supabase/server";

export async function POST(request: NextRequest) {
  try {
    // 1. 解析请求
    const body = await request.json();
    const { param1, param2 } = body;

    // 2. 参数验证
    if (!param1) {
      return NextResponse.json(
        { error: "缺少必需参数" },
        { status: 400 }
      );
    }

    // 3. 业务逻辑
    const supabase = createClient();
    const { data, error } = await supabase
      .from("table_name")
      .select("*")
      .eq("id", param1)
      .single();

    // 4. 错误处理
    if (error) {
      console.error("[API] Database error:", error);
      return NextResponse.json(
        { error: "服务错误" },
        { status: 500 }
      );
    }

    // 5. 返回响应
    return NextResponse.json({ data });
  } catch (error) {
    console.error("[API] Unexpected error:", error);
    return NextResponse.json(
      { error: "服务器错误" },
      { status: 500 }
    );
  }
}
```

### 2.3 Hook 模板

```typescript
/**
 * Hook 说明
 *
 * @param param1 - 参数1说明
 * @returns 返回值说明
 */

import { useState, useEffect, useCallback } from "react";

interface UseHookResult {
  data: any;
  loading: boolean;
  error: string | null;
  refetch: () => void;
}

export function useHook(param1: string): UseHookResult {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  const fetchData = useCallback(async () => {
    setLoading(true);
    setError(null);

    try {
      // 数据获取逻辑
      const response = await fetch(`/api/data?param1=${param1}`);
      const result = await response.json();
      setData(result);
    } catch (err) {
      setError("获取数据失败");
    } finally {
      setLoading(false);
    }
  }, [param1]);

  useEffect(() => {
    fetchData();
  }, [fetchData]);

  return {
    data,
    loading,
    error,
    refetch: fetchData,
  };
}
```

## 三、安全规范

### 3.1 输入验证

**验证清单：**
```typescript
// 1. 类型检查
if (typeof input !== "string") {
  return { error: "参数类型错误" };
}

// 2. 长度检查
if (input.length < 5 || input.length > 1000) {
  return { error: "参数长度错误" };
}

// 3. 格式检查（如邮箱）
if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)) {
  return { error: "邮箱格式错误" };
}

// 4. 内容检查（如 XSS）
if (/<script|javascript:|onerror=/i.test(input)) {
  return { error: "内容包含非法字符" };
}

// 5. 业务逻辑检查
if (businessRule(input)) {
  return { error: "业务规则验证失败" };
}
```

### 3.2 超时控制

**所有外部请求必须有超时：**
```typescript
// ✅ 好的：有超时
const response = await fetch(url, {
  signal: AbortSignal.timeout(60000),
});

// ❌ 坏的：无超时
const response = await fetch(url);
```

### 3.3 速率限制

**匿名用户必须有速率限制：**
```typescript
// 检查速率限制
const rateLimitResult = await checkRateLimit(supabase, clientIp);

if (!rateLimitResult.allowed) {
  return NextResponse.json(
    {
      error: "RATE_LIMIT_EXCEEDED",
      message: "请求过于频繁，请稍后再试",
      resetAt: rateLimitResult.resetAt,
    },
    { status: 429, headers: { "Retry-After": "3600" } }
  );
}
```

### 3.4 错误处理

**原则：** 友好提示，不暴露技术细节

```typescript
// ❌ 坏的错误提示
throw new Error("TimeoutError: Request timeout after 60000ms");

// ✅ 好的错误提示
throw new Error("服务响应时间过长，请稍后再试。");
```

### 3.5 优雅降级

**外部服务失败时的处理：**
```typescript
// Redis 不可用
const redisAvailable = await isRedisAvailable();
if (!redisAvailable) {
  console.warn("[Redis] Unavailable, using fallback");
  // 继续执行，不阻塞
}

// Dify 不可用
try {
  const result = await difyClient.call(params);
  return result;
} catch (error) {
  console.error("[Dify] Call failed:", error);
  // 返回友好错误，不崩溃
  return { error: "AI 服务暂时不可用" };
}
```

## 四、测试策略

### 4.1 单元测试

**测试框架：** Vitest

```typescript
import { describe, it, expect } from "vitest";
import { validateInput } from "./validator";

describe("validateInput", () => {
  it("should reject empty input", () => {
    const result = validateInput("");
    expect(result.valid).toBe(false);
    expect(result.error).toContain("不能为空");
  });

  it("should reject input that is too short", () => {
    const result = validateInput("ab");
    expect(result.valid).toBe(false);
    expect(result.error).toContain("至少5个字符");
  });

  it("should accept valid input", () => {
    const result = validateInput("hello world");
    expect(result.valid).toBe(true);
  });
});
```

### 4.2 集成测试

```typescript
import { describe, it, expect } from "vitest";
import { POST } from "./route";

describe("API Route", () => {
  it("should return 400 for missing parameters", async () => {
    const request = new Request("http://localhost:3000/api/test", {
      method: "POST",
      body: JSON.stringify({}),
    });

    const response = await POST(request);
    expect(response.status).toBe(400);
  });

  it("should return data for valid request", async () => {
    const request = new Request("http://localhost:3000/api/test", {
      method: "POST",
      body: JSON.stringify({ param1: "value" }),
    });

    const response = await POST(request);
    expect(response.status).toBe(200);
    const data = await response.json();
    expect(data).toHaveProperty("data");
  });
});
```

## 六、常见问题

### 6.1 开发问题

**Q: 为什么有些组件没有类型？**
```
A: 历史遗留代码，正在逐步补充。
   优先级：中
   计划：Q2 2026 完成
```

**Q: 为什么有些地方用 `any`？**
```
A: 某些第三方库没有类型定义。
   解决方案：逐步替换为具体类型
```

**Q: 为什么测试覆盖率低？**
```
A: 历史代码没有测试。
   计划：新代码必须有测试
   优先级：高
```

### 6.2 部署问题

**Q: Vercel 部署失败怎么办？**
```bash
1. 检查构建日志
2. 确认环境变量配置
3. 本地构建验证（npm run build）
4. 查看 Vercel 文档
```

**Q: Supabase 连接失败？**
```bash
1. 检查网络连接
2. 验证 API 密钥
3. 查看 Supabase 状态页
4. 联系 Supabase 支持
```

**Q: Redis 连接超时？**
```bash
1. 检查 Redis 状态
2. 验证连接字符串
3. 检查防火墙设置
4. 尝试重启 Redis
```

### 6.3 性能问题

**Q: 页面加载慢怎么办？**
```
优化方案：
1. 检查 bundle 大小
2. 启用代码分割
3. 优化图片（使用 WebP）
4. 启用 CDN
```

**Q: API 响应慢？**
```
优化方案：
1. 检查数据库查询
2. 添加索引
3. 启用 Redis 缓存
4. 优化 Dify 工作流
```
